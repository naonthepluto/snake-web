# Змейка — CLAUDE.md

Игра «Змейка» на Python + pygame. Автор: Лёша Карпич, 5-б.

## Запуск

```
python main.py
```

Требует pygame. Версия Python зафиксирована в `.python-version`.

## Структура файлов

```
snake/
  main.py        — весь игровой код
  VERSION        — текущая версия (читается автоматически при запуске)
  CHANGELOG.md   — история правок
  CLAUDE.md      — этот файл
  assets/        — ресурсы (пока не используются)
  src/           — дополнительный код (пока не используется)
```

## Архитектура главного цикла

Цикл работает в **60 fps** (`clock.tick(60)`). Движение змейки управляется
таймером (`last_tick_ms`), что позволяет анимациям быть плавными независимо
от игровой скорости.

## Игровые объекты

| Объект | Цвет | Условие появления | Эффект |
|--------|------|-------------------|--------|
| Яблоко (красный) | (220,40,40) | всегда 1 на поле | +1 очко, рост, +0.3fps |
| Апельсин (жёлтый) | (255,190,0) | каждые randint(2,4) яблока, 1 на поле | +1 очко, рост, скорость ×⅔ на 3с |
| Пирог (голубой) | (0,200,255) | каждые randint(5,9) яблок, накапливается | очки ×2, рост до 2× текущей длины |

### Механика пирога (grow_pending)
При съедании пирога: `grow_pending += pre_len - 1` (где `pre_len` — длина до вставки головы).
На каждом обычном тике (без еды) если `grow_pending > 0`: хвост не удаляется,
`grow_pending -= 1`. Итого змейка вырастает с `pre_len` до `2 × pre_len`.

### Спаун объектов
- Апельсин: `apples_eaten == next_orange_at`, где `next_orange_at` стартует в `randint(2,4)`
  и после каждого спауна сдвигается на `+randint(2,4)` — всегда 1 на поле (перемещается)
- Пирог: `apples_eaten == next_cake_at`, где `next_cake_at` стартует в `randint(5,9)`
  и сдвигается на `+randint(5,9)` — добавляется в список `cakes[]`, не исчезает
- Если оба триггера совпали: апельсин спаунится первым, пирог избегает его позицию

## Пауза

**ПРОБЕЛ** во время игры переключает паузу (`paused = True/False`).

Когда `paused = True`:
- Игровой тик не выполняется
- Таймер эффекта апельсина не тикает
- Экран рендерится в цвете, затем `pygame.transform.grayscale(screen)` заменяет картинку чёрно-белой
- Поверх рисуется надпись "ПАУЗА" с плавной пульсацией яркости:
  `brightness = 130 + 125 × (sin(t × π × 1.5) + 1) / 2` → ~0.75 Гц, серый ↔ белый

При снятии паузы: `last_tick_ms = now` — сброс таймера тика, чтобы змейка
не сделала несколько ходов сразу для «наверстывания» пропущенного времени.

## Анимация статуса

`make_status_surf(elapsed_ms, text, base_color)` — универсальная функция.
Принимает насыщенный цвет и автоматически строит 3-фазную анимацию:
1. **Вспышка** (0–0.5с): рост 40→100%, тёмный → белый
2. **Дрожание** (0.5–1.5с): насыщенный ↔ белый (~2.5 Гц)
3. **Угасание** (1.5–2.5с): белый → тёмный, alpha 255→0

| Событие | Текст | Цвет |
|---------|-------|------|
| Яблоко | Восторг! | (220,40,40) красный |
| Апельсин | Перевариваю! | (210,150,0) золотой |
| Пирог | Ой! Я съела нельзя.. | (0,190,235) голубой |

## Версионирование

Версия хранится в `VERSION` и читается при старте. При обновлении: менять `VERSION`
и добавлять запись в `CHANGELOG.md`.
